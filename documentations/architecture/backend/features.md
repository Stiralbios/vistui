# Vistui Backend Features

Core domain features of the Vistui backend.

## User

Human users of the system.

### Business Rules

- Identified by email and authenticated with Argon2-hashed passwords.
- JWT tokens are used for session management via OAuth2 Password Bearer flow.
- All authenticated users have the same rights in v1.

### Files

`sources/backend/user/`

## ChatGroup

A grouping created by a user to hold shared memory across one or more chats. It is the primary unit of memory scoping and configuration.

### Business Rules

- Owned by a User.
- Contains config, prompts, and `ranking_formula`.
- At creation, config, prompts, and ranking formula are copied from system defaults unless explicitly supplied.
- Deleting a ChatGroup cascades to Chats, Messages, Events, Topics, and Facts.

### Files

`sources/backend/chatgroup/`

## Chat

Represents a chat window with a chatbot. Each Chat belongs to exactly one ChatGroup.

### Business Rules

- `chatgroup_id` is immutable after creation.
- If a caller creates a Chat without specifying a ChatGroup, the API creates a dedicated ChatGroup with matching name and description.
- Deleting a Chat cascades to Messages, Events, and join links. Topics and Facts in the parent ChatGroup remain unless no other Chat references them via links.

### Files

`sources/backend/chat/`

## Message

A single message inside a Chat, stored as a linked list.

### Business Rules

- `id` is caller-provided to enable idempotent `PUT` creation and editing.
- `prev_message_id` and `next_message_id` must form a coherent sequence per Chat.
- `role` must be one of `system`, `user`, or `assistant`.
- `full_text` has a maximum length of 32,000 characters.
- Embedding, keywords, and salience are recomputed on every save.
- Batch processing state is reset to `waiting` for embedding/salience on every save, and for event/topic/fact only if the message is outside the protected window.

### Files

`sources/backend/message/`

## Event

A time-bounded summary of contiguous messages within a Chat.

### Business Rules

- Scoped to a Chat.
- A single message can contribute to multiple events if the LLM decides it spans event boundaries.
- `embedding` and `salience_score` are set to `null` when rewritten and later regenerated by batch jobs.

### Files

`sources/backend/event/`

## Topic

A summary of a recurring theme across non-contiguous messages in a ChatGroup.

### Business Rules

- Scoped to a ChatGroup.
- Links to non-contiguous Messages via `topic_message_links`.
- When updated, only the latest linked message is the active context; older links form history.

### Files

`sources/backend/topic/`

## Fact

A small, persistent fact to remember across a ChatGroup.

### Business Rules

- Scoped to a ChatGroup.
- Links to multiple Messages via `fact_message_links`.
- The latest linked message is the active context; older links are history.
- Rewritten in place when the underlying truth changes (e.g., "Mom is sick" → "Mom is on vacation").

### Files

`sources/backend/fact/`

## Retrieval

Memory retrieval for new or edited messages.

### Business Rules

- Combines vector search, BM25 search, configurable ranking formula, and token-budget packing.
- Memory ratio is generated by LLM or taken from ChatGroup config.
- Returns a `MemoryResponse` block when `retrieve=true`.

### Files

`sources/backend/retrieval/`

## Batch

Asynchronous background worker for embeddings, salience, and consolidation.

### Business Rules

- Single in-process APScheduler worker.
- Starts after a period of inactivity.
- Processes messages in chronological order, one sub-task at a time.
- Persists processing state in the database for resume on restart.

### Files

`sources/backend/batch/`

## Common Patterns

All features follow the same architecture:

- Feature-based folder layout.
- CRUD endpoints with lightweight auth.
- Internal/external schema split with `context` for service logic.
- Stateless stores using `@with_async_session`.
- Services that may call multiple stores but never other services.
- Full test coverage with polyfactory factories.
